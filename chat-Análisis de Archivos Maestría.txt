## 🎯 ¿Por qué Django es mejor que solo Power BI para este proyecto?

| Criterio | Power BI | Django + Python |
|----------|----------|-----------------|
| **Modelos predictivos reales** | Limitado (solo Python visual básico) | ✅ Scikit-learn, TensorFlow, Prophet completos |
| **APIs para integración futura** | ❌ No tiene | ✅ REST API nativa (Django REST Framework) |
| **Autenticación multiusuario** | Solo con Power BI Service | ✅ Django Auth + roles |
| **Automatización ETL programada** | Requiere Gateway | ✅ Celery + Redis nativo |
| **Portabilidad del producto** | Licencia Microsoft $$$ | ✅ Open source, desplegable en cualquier lado |
| **Alineación con Big Data** | Visualización | ✅ Pipeline completo: ingesta → ML → visualización |
| **Valor profesional/académico** | Medio | ✅ Muy alto (full-stack data product) |

**Conclusión:** Power BI es excelente para visualización, pero **Django convierte tu proyecto en una plataforma de Big Data real**. Lo ideal sería **ambos**: Django como backend + Power BI consumiendo la API REST de Django.

"""Django's command-line utility for administrative tasks."""
import os
import sys


def main():
    """Run administrative tasks."""
    os.environ.setdefault('DJANGO_SETTINGS_MODULE', 'config.settings.dev')
    try:
        from django.core.management import execute_from_command_line
    except ImportError as exc:
        raise ImportError(
            "Couldn't import Django. Are you sure it's installed and "
            "available on your PYTHONPATH environment variable? Did you "

if __name__ == '__main__':

from django.contrib.gis.geos import Point, GEOSGeometry
from django.contrib.gis.measure import D
import logging

logger = logging.getLogger(__name__)


def create_point_from_coords(lat, lng, srid=4326):
    """Crea un PointField desde coordenadas lat/lng"""
    try:
        return Point(float(lng), float(lat), srid=srid)
    except (ValueError, TypeError) as e:
        logger.error(f"Error creating point from ({lat}, {lng}): {e}")
        return None


def get_nearby_stations(model_class, point, radius_km=50):
    """Obtiene estaciones dentro de un radio"""
    return model_class.objects.filter(
        location__distance_lte=(point, D(km=radius_km))
    ).annotate(
        distance=models.functions.Distance('location', point)
    ).order_by('distance')


def calculate_distance(point1, point2):
    """Calcula distancia entre dos puntos en km"""
    return point1.distance(point2) * 111.32  # Aproximación en km


def format_geojson_queryset(queryset, geometry_field='location'):
    """Convierte un queryset geoespacial a GeoJSON"""
    from django.contrib.gis.serializers import serialize
    return serialize('geojson', queryset, 
                     geometry_field=geometry_field,
                     fields=('name', 'address', 'state'))

"""
Comando para inspeccionar la BD existente y generar modelos Django
"""
from django.core.management.base import BaseCommand
from django.db import connection


class Command(BaseCommand):
    help = 'Inspecciona la BD truck_routes_db existente y lista todas las tablas'
    
    def handle(self, *args, **options):
        self.stdout.write(self.style.SUCCESS('=== Inspeccionando BD truck_routes_db ==='))
        
        with connection.cursor() as cursor:
            # Listar todas las tablas
            cursor.execute("""
                SELECT table_name 
                FROM information_schema.tables 
                WHERE table_schema = 'public'
                ORDER BY table_name;
            """)
            tables = cursor.fetchall()
            
            self.stdout.write(f'\n📊 Total de tablas encontradas: {len(tables)}\n')
            
            for table in tables:
                table_name = table[0]
                self.stdout.write(f'  📋 {table_name}')
                
                # Contar registros
                cursor.execute(f'SELECT COUNT(*) FROM "{table_name}";')
                count = cursor.fetchone()[0]
                self.stdout.write(f'     → {count} registros')
            
            # Listar tablas con geometría
            cursor.execute("""
                SELECT f_table_name, f_geometry_column, type, srid
                FROM geometry_columns;
            """)
            geo_tables = cursor.fetchall()
            
            self.stdout.write(f'\n🗺️  Tablas con datos geoespaciales: {len(geo_tables)}\n')
            for geo in geo_tables:
                self.stdout.write(f'  📍 {geo[0]}.{geo[1]} ({geo[2]}, SRID:{geo[3]})')
        
        self.stdout.write(self.style.SUCCESS('\n✅ Inspección completada'))

    has_restaurant = models.BooleanField(default=False, verbose_name='Restaurante')
    has_wifi = models.BooleanField(default=False, verbose_name='WiFi')
    has_showers = models.BooleanField(default=False, verbose_name='Duchas')
    has_tires = models.BooleanField(default=False, verbose_name='Llantas')
    has_mechanic = models.BooleanField(default=False, verbose_name='Mecánico')
    
    # Capacidad
    truck_parking_spaces = models.IntegerField(null=True, blank=True, 
                                                verbose_name='Espacios para camiones')

    source_file = models.CharField(max_length=255, blank=True, 
                                   verbose_name='Archivo fuente')
    original_id = models.CharField(max_length=100, blank=True, 
                                   verbose_name='ID original')

        db_table = 'truck_stations'
        verbose_name = 'Estación de Descanso'
        verbose_name_plural = 'Estaciones de Descanso'
        indexes = [
            models.Index(fields=['operator']),
            models.Index(fields=['state']),
        ]
    
    def __str__(self):
        return f"{self.name} ({self.operator}) - {self.state}"
    
    @property
    def services_list(self):
        """Lista de servicios disponibles"""
        services = []
        if self.has_diesel: services.append('Diésel')
        if self.has_def: services.append('DEF')
        if self.has_restaurant: services.append('Restaurante')
        if self.has_wifi: services.append('WiFi')
        if self.has_showers: services.append('Duchas')
        if self.has_tires: services.append('Llantas')
        if self.has_mechanic: services.append('Mecánico')
        return services

from django.contrib.gis.geos import Point
from django.contrib.gis.measure import D

from .models import TruckStation, TruckStopParking
from .serializers import (
    TruckStationSerializer, TruckStationListSerializer,
    TruckStopParkingSerializer
)

class TruckStationViewSet(viewsets.ReadOnlyModelViewSet):
    """API endpoint para estaciones de descanso"""
    queryset = TruckStation.objects.filter(is_active=True)
    filter_backends = [DjangoFilterBackend, filters.SearchFilter, filters.OrderingFilter]
    filterset_fields = ['operator', 'state', 'has_diesel', 'has_showers', 'has_tires']
    search_fields = ['name', 'operator', 'address', 'city']
    ordering_fields = ['name', 'state', 'created_at']
    
    def get_serializer_class(self):
        if self.action == 'list':
            return TruckStationListSerializer
        return TruckStationSerializer
    
    @action(detail=False, methods=['get'])
    def nearby(self, request):
        """Busca estaciones cercanas a un punto"""
        lat = request.query_params.get('lat')
        lng = request.query_params.get('lng')
        radius = request.query_params.get('radius', 50)
        
        if not lat or not lng:
            return Response({'error': 'Se requieren parámetros lat y lng'}, status=400)
        
        point = Point(float(lng), float(lat), srid=4326)
        stations = TruckStation.objects.filter(
            location__distance_lte=(point, D(km=float(radius)))
        )[:20]
        
        serializer = TruckStationListSerializer(stations, many=True)
        return Response(serializer.data)
    
    @action(detail=False, methods=['get'])
    def stats(self, request):
        """Estadísticas de estaciones"""
        total = TruckStation.objects.count()
        by_operator = TruckStation.objects.values('operator').annotate(
            count=models.Count('id')
        ).order_by('-count')
        
        return Response({
            'total': total,
            'by_operator': list(by_operator)
        })

    search_fields = ['name', 'address']

        if services_required:

import logging

logger = logging.getLogger(__name__)

def create_point_from_coords(lat, lng, srid=4326):
    """Crea un PointField desde coordenadas lat/lng"""
    try:
        return Point(float(lng), float(lat), srid=srid)
    except (ValueError, TypeError) as e:
        logger.error(f"Error creating point from ({lat}, {lng}): {e}")
        return None

def get_nearby(model_class, point, radius_km=50):
    """Obtiene entidades dentro de un radio"""
    return model_class.objects.filter(
        location__distance_lte=(point, D(km=radius_km))
    )

from django.conf import settings

class TireWearPredictor:
    """Predice el desgaste de llantas basado en distancia y tipo de ruta"""
    
    def __init__(self):
        self.model_path = Path(settings.ML_MODELS_DIR) / 'tire_wear_model.pkl'
        self.model = None
        if self.model_path.exists():
            self.model = joblib.load(self.model_path)
    
    def train(self, X, y):
        """Entrena el modelo con datos históricos"""
        from sklearn.ensemble import RandomForestRegressor
        self.model = RandomForestRegressor(n_estimators=100, random_state=42)
        self.model.fit(X, y)
        joblib.dump(self.model, self.model_path)
        return self.model
    
    def predict(self, distance_km, route_type, truck_weight_tons):
        """Predice desgaste"""
        if not self.model:
            return None
        X = np.array([[distance_km, route_type, truck_weight_tons]])
        return self.model.predict(X)[0]

## 🧠 ESPECIFICACIONES DE MACHINE LEARNING

### Modelo 1: TireWearPredictor
- **Input:** distance_km, route_type, truck_weight_tons, weather_condition
- **Output:** wear_score (0-100)
- **Algoritmo:** RandomForestRegressor
- **Métricas:** R² > 0.75, MAE < 10

### Modelo 2: DemandForecaster
- **Input:** region, month, carrier_count, traffic_volume
- **Output:** predicted_demand (number of service requests)
- **Algoritmo:** GradientBoostingRegressor
- **Métricas:** R² > 0.80, MAPE < 15%

### Modelo 3: CoverageOptimizer
- **Input:** coordinates of existing stations, population density, traffic data
- **Output:** recommended_locations (list of coordinates)
- **Algoritmo:** K-Means clustering + gap analysis
- **Métrica:** Silhouette Score > 0.6

Todos los modelos deben:
- Guardarse en ml_models/ como archivos .pkl con joblib
- Incluir validación de datos de entrada
- Retornar métricas de rendimiento después del entrenamiento
- Ser recargables sin reiniciar Django

- **3 Modelos ML completos:**

#### Modelo 1: TireWearPredictor
- **Input:** distance_km, route_type, truck_weight_tons, weather_condition
- **Output:** wear_score (0-100)
- **Algoritmo:** RandomForestRegressor
- **Métricas:** R² > 0.75, MAE < 10
- **Validación:** Cross-validation 5-fold

#### Modelo 2: DemandForecaster
- **Input:** region, month, carrier_count, traffic_volume
- **Output:** predicted_demand (número de solicitudes de servicio)
- **Algoritmo:** GradientBoostingRegressor
- **Métricas:** R² > 0.80, MAPE < 15%

#### Modelo 3: CoverageOptimizer
- **Input:** coordenadas de estaciones existentes, densidad poblacional, tráfico
- **Output:** recommended_locations (lista de coordenadas para nuevas estaciones)
- **Algoritmo:** K-Means clustering + gap analysis
- **Métrica:** Silhouette Score > 0.6

- **Validadores:**
  - `validate_prediction()` - Valida coherencia de predicciones
  - `validate_model_performance()` - Valida métricas (R², MAE, RMSE)

## 🗄️ ESPECIFICACIONES DE LA BASE DE DATOS

PostgreSQL 16 + PostGIS 3.4. Base de datos: `truck_routes_db`

### Tablas principales:

#### 🆕 TABLAS IoT (CRÍTICAS):

1. **devices** - Dispositivos PIN instalados
   - id, device_code (unique), truck_id, installed_at
   - status (active/inactive/maintenance)
   - firmware_version, last_heartbeat
   - location (PointField SRID 4326)

2. **sensor_data** - Datos de sensores (time-series)
   - id, device (FK), timestamp
   - tire_pressure_fl/fr/rl/rr (psi)
   - tire_temp_fl/fr/rl/rr (°F)
   - latitude, longitude, speed_mph, heading
   - accelerometer_x/y/z
   - battery_level, signal_strength
   - INDEX en (device, timestamp) para queries rápidos

3. **alerts** - Alertas generadas
   - id, device (FK), alert_type (LOW_PRESSURE, HIGH_TEMP, PUNCTURE, etc.)
   - severity (LOW, MEDIUM, HIGH, CRITICAL)
   - message, tire_position (FL, FR, RL, RR)
   - location (PointField), timestamp
   - status (ACTIVE, ACKNOWLEDGED, RESOLVED)
   - resolved_at, resolution_notes

4. **recommendations** - Recomendaciones generadas
   - id, alert (FK, nullable), device (FK)
   - recommendation_type (TIRE_SHOP, REST_STOP, RESTAURANT, RECYCLING)
   - target_location (FK a estación/tienda/centro)
   - distance_km, estimated_time_min
   - priority, created_at
   - accepted_by_driver (boolean), accepted_at

#### TABLAS DE INFRAESTRUCTURA:

5. **truck_stations** - Estaciones PILOT/LOVES/TravelCenters
   - name, operator, brand, phone, website, address, city, state
   - has_diesel, has_def, has_restaurant, has_wifi, has_showers, has_tires, has_mechanic
   - truck_parking_spaces, location (PointField)

6. **highway_routes** - Rutas NHS
   - route_id, route_name, route_type, state, length_km
   - route_geometry (LineStringField)

7. **alternative_fuel_stations** - Combustible alternativo
   - name, fuel_type, is_hd_fuel, access_type, operators, location

8. **tire_shops** - Tiendas de llantas
   - name, shop_type, specializes_trucks, phone, location

9. **tire_recycling_centers** - Centros de reciclaje EPA
   - name, capacity_tons, recycling_type, location

10. **wim_stations** - Estaciones WIM
    - name, station_code, direction, location

11. **size_weight_regulations** - Regulaciones por estado
    - state, max_weight_lbs, max_length_ft, max_width_in, max_height_ft

12. **carriers** - Transportistas FMCSA
    - dot_number, legal_name, num_power_units, num_drivers, state

13. **monthly_transport_indicators** - Estadísticas BTS
    - date, indicator_name, value, unit, region

14. **pois** - Puntos de interés (restaurantes, hoteles, cafés)
    - name, category, phone, website, location

15. **predictions** - Predicciones ML guardadas
    - model_name, prediction_type, device_id, region
    - predicted_value, confidence, features_json, created_at

### 📌 APP: analytics (🆕 MEJORADA)
- **5 Modelos ML completos:**

#### Modelo 1: TireFailurePredictor (🆕 CRÍTICO)
- **Input:** historical_sensor_data (presión, temperatura, vibración, velocidad, distancia)
- **Output:** probability_of_failure_next_7_days (0-100%)
- **Algoritmo:** LSTM (TensorFlow) + XGBoost ensemble
- **Métricas:** AUC-ROC > 0.85, F1-Score > 0.80
- **Entrenamiento:** Datos históricos de sensores + registros de fallos

#### Modelo 2: TireWearEstimator
- **Input:** distance_km, route_type, truck_weight_tons, avg_speed, road_condition
- **Output:** estimated_wear_percentage (0-100%)
- **Algoritmo:** RandomForestRegressor
- **Métricas:** R² > 0.75, MAE < 5%

#### Modelo 3: DemandForecaster
- **Input:** region, month, carrier_count, traffic_volume, season
- **Output:** predicted_demand (número de servicios necesarios)
- **Algoritmo:** GradientBoostingRegressor
- **Métricas:** R² > 0.80, MAPE < 15%

#### Modelo 4: OptimalLocationRecommender
- **Input:** existing_stations_coords, population_density, traffic_data, demand_forecast
- **Output:** recommended_locations (lista de coordenadas para nuevas estaciones)
- **Algoritmo:** K-Means clustering + gap analysis + genetic algorithm
- **Métrica:** Silhouette Score > 0.6, coverage improvement > 20%

#### Modelo 5: RouteRiskScorer (🆕 NUEVO)
- **Input:** route_geometry, weather_data, road_condition, historical_accidents
- **Output:** risk_score (0-100), risk_factors (lista)
- **Algoritmo:** GradientBoostingClassifier
- **Métricas:** AUC-ROC > 0.80

- **Validadores:**
  - `validate_prediction()` - Valida coherencia
  - `validate_model_performance()` - Valida métricas
- Comando: `entrenar_modelos` (entrena los 5 modelos)

## 🔍 PRIMERO: Aclaración CRÍTICA sobre la Arquitectura

Tu comentario revela algo **fundamental** que cambia todo el diseño. Hay **3 posibles arquitecturas** para el dispositivo PIN:

### ❌ Opción 1: 18 dispositivos independientes por tractomula (NO recomendada)
```
Tractomula → 18 dispositivos PIN (cada uno con GPS, batería, SIM card)
```
**Problemas:**
- Costo brutal: 18 GPS + 18 SIM cards + 18 baterías por camión
- Redundancia de datos (el GPS es el mismo para todo el camión)
- Complejidad de sincronización
- Consumo de datos móviles x18

### ⚠️ Opción 2: 1 dispositivo maestro + 18 sensores esclavos (MEJOR OPCIÓN) ✅
```
Tractomula → 1 dispositivo PIN MAESTRO (GPS, SIM, CPU, batería)
             └── 18 sensores esclavos (1 por llanta: presión, temperatura, vibración)
             └── Comunicación interna: CAN Bus o Bluetooth Mesh
```
**Ventajas:**
- 1 solo GPS, 1 sola SIM card, 1 sola batería grande
- Los 18 sensores son baratos (solo miden presión/temp/vibración)
- Sincronización perfecta (todos con el mismo timestamp del maestro)
- Escalable (puedes agregar más sensores fácilmente)
- **Es la arquitectura estándar en la industria** (TPMS - Tire Pressure Monitoring Systems)

### ⚠️ Opción 3: 1 dispositivo por vehículo que lee 18 sensores cableados
```
Tractomula → 1 dispositivo PIN (con 18 entradas cableadas)
```
**Problema:** Los cables en llantas que giran y se flexionan se rompen. No viable.

### 📦 Componentes del Simulador:

```python
# simulador_pin.py - Genera datos realistas de 1 tractomula con 18 sensores

class TractomulaSimulator:
    """Simula una tractomula completa con 18 sensores PIN"""
    
    def __init__(self, truck_id):
        self.truck_id = truck_id
        self.master_device = MasterDevice(truck_id)  # GPS, velocidad, etc.
        self.tire_sensors = [
            TireSensor("FL-1"), TireSensor("FL-2"),
            TireSensor("FR-1"), TireSensor("FR-2"),
            TireSensor("RL-1"), TireSensor("RL-2"),
            TireSensor("RL-3"), TireSensor("RL-4"),
            TireSensor("RR-1"), TireSensor("RR-2"),
            TireSensor("RR-3"), TireSensor("RR-4"),
            TireSensor("TL-1"), TireSensor("TL-2"),
            TireSensor("TL-3"), TireSensor("TL-4"),
            TireSensor("TR-1"), TireSensor("TR-2"),
        ]
    
    def generar_lectura(self):
        """Genera una lectura coherente cada 30 segundos"""
        # 1. Datos del maestro (comunes a todo el camión)
        master_data = self.master_device.read()  # GPS, velocidad, etc.
        
        # 2. Datos de cada llanta (correlacionados entre sí)
        tires_data = []
        for sensor in self.tire_sensors:
            # La presión y temperatura varían según posición y uso
            tire_data = sensor.read(
                speed=master_data.speed,
                ambient_temp=master_data.ambient_temp,
                road_condition=master_data.road_condition
            )
            tires_data.append(tire_data)
        
        # 3. Paquete completo para enviar a la plataforma
        return {
            "truck_id": self.truck_id,
            "timestamp": datetime.now().isoformat(),
            "master": master_data,
            "tires": tires_data
        }
```

## 🧠 MODELOS DE ML (5 MODELOS)

### Modelo 1: TireFailurePredictor (LSTM + XGBoost)
- **Input:** Últimas 24h de lecturas del sensor (presión, temp, vibración)
- **Output:** Probabilidad de fallo en próximos 7 días (0-100%)
- **Entrenamiento:** Datos simulados con escenarios de fallo inyectados
- **Métricas:** AUC-ROC > 0.85

### Modelo 2: TireWearEstimator (RandomForest)
- **Input:** km acumulados, posición, presión promedio, temp promedio, tipo de ruta
- **Output:** Porcentaje de desgaste (0-100%)
- **Métricas:** R² > 0.75

### Modelo 3: DemandForecaster (GradientBoosting)
- **Input:** Región, mes, número de camiones activos, tráfico
- **Output:** Demanda predicha de servicios
- **Métricas:** MAPE < 15%

### Modelo 4: OptimalLocationRecommender (K-Means + Gap Analysis)
- **Input:** Ubicación de estaciones existentes, densidad de tráfico, demanda
- **Output:** Coordenadas recomendadas para nuevas estaciones
- **Métrica:** Silhouette Score > 0.6

### Modelo 5: RouteRiskScorer (GradientBoosting Classifier)
- **Input:** Geometría de ruta, clima, historial de accidentes, tipo de carretera
- **Output:** Risk score (0-100) + factores de riesgo
- **Métricas:** AUC-ROC > 0.80

El sistema DEBE monitorear y alertar sobre estas regulaciones para conductores de carga (property-carrying):

| **Sleeper Berth** | Puede dividir 10h en: 7h en sleeper + 2h off (o 2h + 7h) | Validar combinación |
| **Adverse Conditions** | +2 horas en condiciones adversas (lluvia, nieve) | Ajustar límites automáticamente |

## 🧮 PARÁMETROS QUE AFECTAN EL DESGASTE DE LLANTAS (MUY IMPORTANTE)

El sistema DEBE considerar TODOS estos parámetros para predecir desgaste con precisión:

### 📊 1. PARÁMETROS FÍSICOS (Medidos por sensores PIN)

| Parámetro | Unidad | Rango Normal | Impacto en Desgaste | Fórmula de Ajuste |
|-----------|--------|--------------|---------------------|-------------------|
| **Presión de inflación** | PSI | 90-110 | Sub-inflación: desgaste en bordes. Sobre-inflación: desgaste en centro | factor_presion = 1 + 0.02 * |PSI_actual - PSI_recomendado| / PSI_recomendado |
| **Temperatura de llanta** | °F | 100-140 | Cada 10°F sobre 120°F = +5% desgaste | factor_temp = 1 + 0.005 * max(0, temp - 120) |
| **Vibración (acelerómetro)** | G | 0.1-0.5 | Vibración alta = desbalanceo o mala suspensión | factor_vib = 1 + 2 * max(0, vib - 0.5) |
| **Velocidad** | MPH | 55-65 | Cada 10 MPH sobre 55 = +10% desgaste | factor_vel = 1 + 0.01 * max(0, speed - 55) |
| **Carga en llanta** | lbs | Variable | Mayor carga = mayor desgaste (exponencial) | factor_carga = (carga_actual / carga_nominal)^1.25 |

### 📊 2. PARÁMETROS OPERACIONALES (De OBD-II y GPS)

| Parámetro | Impacto en Desgaste | Cómo se mide |
|-----------|---------------------|--------------|
| **Kilometraje acumulado** | Desgaste lineal base | Odometer del vehículo |
| **Estilo de conducción** | Frenazos = +20% desgaste | Acelerómetro (frenadas > 0.3G) |
| **Aceleraciones bruscas** | +15% desgaste | Acelerómetro (aceleraciones > 0.2G) |
| **Tiempo en ralentí** | Calienta llantas sin desgaste útil | RPM del motor + velocidad = 0 |
| **RPM del motor** | Alto RPM = mayor torque = mayor desgaste | OBD-II |

### 📊 3. PARÁMETROS AMBIENTALES (APIs externas)

| Parámetro | Impacto | Fuente de datos |
|-----------|---------|-----------------|
| **Temperatura ambiente** | Afecta presión y compuesto de goma | API de clima (OpenWeatherMap) |
| **Humedad** | Afecta tracción y temperatura | API de clima |
| **Lluvia/Nieve** | +10% desgaste por menor tracción | API de clima |
| **Viento** | Afecta estabilidad y desgaste lateral | API de clima |
| **Altitud** | Afecta presión de aire | GPS del dispositivo |

### 📊 4. PARÁMETROS DE RUTA (Geoespaciales)

| Parámetro | Impacto | Cómo se calcula |
|-----------|---------|-----------------|
| **Tipo de superficie** | Grava = +30% desgaste vs asfalto | Mapa de tipo de carretera |
| **Pendiente** | Subidas = +15% desgaste en drive tires | Elevación GPS |
| **Curvas** | Curvas cerradas = +20% desgaste lateral | Geometría de ruta |
| **Calidad del pavimento** | Baches = +25% desgaste | Base de datos de DOT |
| **Zona urbana vs rural** | Ciudad = +20% desgaste (más frenados) | Clasificación de vía |

### 📊 5. PARÁMETROS DE LA LLANTA (Configuración)

| Parámetro | Impacto | Valor típico |
|-----------|---------|--------------|
| **Tipo de llanta** | Long-haul vs regional vs all-weather | Configuración inicial |
| **Marca y modelo** | Diferentes compuestos | Configuración inicial |
| **Edad de la llanta** | Degradación del compuesto | Fecha de instalación |
| **Historial de reparaciones** | Parches reducen vida útil | Registro de mantenimiento |
| **Posición en el vehículo** | Steer vs drive vs trailer | Configuración fija |
| **Profundidad inicial de banda** | 32/32" nueva | Configuración inicial |

### 📊 6. PARÁMETROS DE MANTENIMIENTO

| Parámetro | Impacto | Frecuencia |
|-----------|---------|------------|
| **Rotación de llantas** | Equaliza desgaste | Cada 10,000-15,000 millas |
| **Alineación** | Desalineación = +50% desgaste irregular | Cada 50,000 millas |
| **Balanceo** | Desbalanceo = desgaste en parches | Cada 20,000 millas |
| **Inspección visual** | Detecta problemas tempranos | Mensual |

## 🧮 FÓRMULA DE PREDICCIÓN DE DESGASTE (IMPLEMENTAR EN ML)

### Fórmula Base:
```
desgaste_32nds = (millas_recibidas / 1000) * tasa_base_desgaste * 
                 factor_presion * factor_temperatura * factor_velocidad * 
                 factor_carga * factor_vibracion * factor_conduccion * 
                 factor_ambiente * factor_ruta * factor_mantenimiento
```

### Factores de Ajuste:
```python
def calcular_factor_presion(psi_actual, psi_recomendado):
    """Sub-inflación es MUY dañina"""
    diff_percent = abs(psi_actual - psi_recomendado) / psi_recomendado
    if psi_actual < psi_recomendado:
        return 1 + (0.02 * diff_percent * 100)  # Sub-inflación es peor
    else:
        return 1 + (0.01 * diff_percent * 100)  # Sobre-inflación es menos dañina

def calcular_factor_temperatura(temp_f):
    """Cada 10°F sobre 120°F = +5% desgaste"""
    return 1 + 0.005 * max(0, temp_f - 120)

def calcular_factor_velocidad(speed_mph):
    """Cada 10 MPH sobre 55 = +10% desgaste"""
    return 1 + 0.01 * max(0, speed_mph - 55)

def calcular_factor_carga(carga_actual_lbs, carga_nominal_lbs):
    """Desgaste ∝ carga^1.25"""
    ratio = carga_actual_lbs / carga_nominal_lbs
    return ratio ** 1.25

def calcular_factor_vibracion(vibration_g):
    """Vibración alta = desbalanceo o problemas"""
    return 1 + 2 * max(0, vibration_g - 0.5)

def calcular_factor_conduccion(num_hard_braking, num_hard_accel, total_miles):
    """Frenazos y aceleraciones bruscas"""
    hard_events_per_100mi = (num_hard_braking + num_hard_accel) / (total_miles / 100)
    return 1 + 0.02 * hard_events_per_100mi

def calcular_factor_ambiente(temperature_ambient_f, rain_mm, wind_mph):
    """Condiciones climáticas"""
    factor = 1.0
    if temperature_ambient_f > 90:
        factor += 0.05  # Calor extremo
    if rain_mm > 10:
        factor += 0.10  # Lluvia
    if wind_mph > 30:
        factor += 0.05  # Viento fuerte
    return factor

def calcular_factor_ruta(road_type, gradient_percent, curvature):
    """Tipo de ruta"""
    factor = 1.0
    if road_type == 'gravel':
        factor += 0.30
    elif road_type == 'urban':
        factor += 0.20
    if abs(gradient_percent) > 5:
        factor += 0.15  # Pendientes pronunciadas
    if curvature > 0.1:  # Curvas cerradas
        factor += 0.20
    return factor

def calcular_factor_mantenimiento(days_since_rotation, days_since_alignment, days_since_balancing):
    """Mantenimiento"""
    factor = 1.0
    if days_since_rotation > 15000:  # millas
        factor += 0.10
    if days_since_alignment > 50000:
        factor += 0.15
    if days_since_balancing > 20000:
        factor += 0.10
    return factor
```

### Predicción de Vida Restante:
```python
def predecir_vida_restante_32nds(sensor, tread_depth_actual_32nds):
    """Predice cuántos 32nds restan antes de llegar al mínimo legal"""
    
    # Obtener configuración de la posición
    config = TirePositionConfig.objects.get(position=sensor.position)
    min_legal_32nds = config.critical_tread_depth_32nds
    
    # Calcular tasa de desgaste actual (últimos 30 días)
    recent_readings = TireReading.objects.filter(
        sensor=sensor,
        timestamp__gte=now() - timedelta(days=30)
    )
    
    # Calcular factores promedio
    avg_pressure = recent_readings.aggregate(Avg('pressure_psi'))['pressure_psi__avg']
    avg_temp = recent_readings.aggregate(Avg('temperature_f'))['temperature_f__avg']
    avg_speed = VehicleReading.objects.filter(
        master_device=sensor.master_device,
        timestamp__gte=now() - timedelta(days=30)
    ).aggregate(Avg('speed_mph'))['speed_mph__avg']
    
    # Obtener peso del eje (de báscula o estimación)
    axle_weight = get_axle_weight(sensor)  # Función auxiliar
    
    # Calcular factores
    factor_presion = calcular_factor_presion(avg_pressure, config.recommended_pressure_psi)
    factor_temp = calcular_factor_temperatura(avg_temp)
    factor_vel = calcular_factor_velocidad(avg_speed)
    factor_carga = calcular_factor_carga(axle_weight / 2, config.max_load_lbs)  # /2 para 2 llantas
    
    # Tasa de desgaste ajustada (32nds por 1000 millas)
    tasa_base = config.base_wear_rate_32nds_per_1000mi
    tasa_ajustada = tasa_base * factor_presion * factor_temp * factor_vel * factor_carga
    
    # Millas restantes estimadas
    tread_remaining = tread_depth_actual_32nds - min_legal_32nds
    miles_remaining = (tread_remaining / tasa_ajustada) * 1000
    
    return {
        'tread_remaining_32nds': tread_remaining,
        'miles_remaining': miles_remaining,
        'wear_rate_32nds_per_1000mi': tasa_ajustada,
        'days_remaining': miles_remaining / avg_daily_miles(sensor.truck),
        'factors': {
            'pressure': factor_presion,
            'temperature': factor_temp,
            'speed': factor_vel,
            'load': factor_carga
        }
    }
```

## 🧠 MODELOS DE ML (6 MODELOS)

### Modelo 1: TireWearPredictor (🆕 MEJORADO)
- **Input:** 20+ parámetros (presión, temp, carga, velocidad, vibración, estilo conducción, clima, ruta, mantenimiento, etc.)
- **Output:** Porcentaje de desgaste actual + predicción de vida restante
- **Algoritmo:** XGBoost Regressor + fórmula física híbrida
- **Métricas:** R² > 0.85, MAE < 5%

### Modelo 2: TireFailurePredictor (LSTM)
- **Input:** Series temporales de 24h (presión, temp, vibración)
- **Output:** Probabilidad de fallo en próximos 7 días
- **Métricas:** AUC-ROC > 0.85

### Modelo 3: HOSCompliancePredictor
- **Input:** Historial de conducción del conductor, ruta planificada, tráfico
- **Output:** Probabilidad de violación HOS en las próximas 2 horas
- **Métricas:** Precision > 0.90, Recall > 0.85

### Modelo 4: OverweightRiskPredictor
- **Input:** Configuración del vehículo, carga declarada, ruta, estado
- **Output:** Probabilidad de overweight + recomendación de redistribución
- **Métricas:** AUC-ROC > 0.80

### Modelo 5: DemandForecaster
- **Input:** Región, mes, número de camiones, tráfico
- **Output:** Demanda predicha de servicios
- **Métricas:** MAPE < 15%

### Modelo 6: OptimalLocationRecommender
- **Input:** Estaciones existentes, densidad tráfico, demanda
- **Output:** Coordenadas recomendadas para nuevas estaciones
- **Métrica:** Silhouette Score > 0.6

### 1. HOURS OF SERVICE (HOS) - FMCSA 49 CFR Part 395
- 11-Hour Driving Limit: Máximo 11h conducción tras 10h off-duty
- 14-Hour Limit: No conducir después de 14h on-duty
- 30-Minute Break: Descanso 30 min tras 8h acumuladas conducción
- 60/70-Hour Limit: 60h en 7 días o 70h en 8 días
- 10-Hour Off-Duty: Mínimo 10h descanso antes de reiniciar ciclo
- Sleeper Berth: Puede dividir 10h en 7h + 2h
- Adverse Conditions: +2h en condiciones adversas
- Short-Haul Exception: Exento si radio 150 air-miles + 14h máximo

### 2. PROFUNDIDAD MÍNIMA DE BANDA - FMCSA 49 CFR 393.75
- Steer (FL, FR): Mínimo legal 4/32", recomendado PIN 6/32"
- Drive (RL, RR): Mínimo legal 2/32", recomendado PIN 4/32"
- Trailer (TL, TR): Mínimo legal 2/32", recomendado PIN 4/32"

### 3. FEDERAL BRIDGE GROSS WEIGHT FORMULA - 23 CFR 658.17
- Peso total vehículo: 80,000 lbs máximo federal
- Eje simple: 20,000 lbs
- Tándem de ejes: 34,000 lbs

## 🧮 FÓRMULA DE PREDICCIÓN DE DESGASTE (IMPLEMENTAR)

```
desgaste_32nds = (millas/1000) * tasa_base * 
                 factor_presion * factor_temperatura * factor_velocidad * 
                 factor_carga * factor_vibracion * factor_conduccion * 
                 factor_ambiente * factor_ruta * factor_mantenimiento
```

### Factores de ajuste:
- factor_presion = 1 + 0.02 * |PSI_actual - PSI_recomendado| / PSI_recomendado
- factor_temperatura = 1 + 0.005 * max(0, temp_f - 120)
- factor_velocidad = 1 + 0.01 * max(0, speed_mph - 55)
- factor_carga = (carga_actual / carga_nominal)^1.25
- factor_vibracion = 1 + 2 * max(0, vibration_g - 0.5)
- factor_conduccion = 1 + 0.02 * hard_events_per_100mi
- factor_ambiente = 1.0 + ajustes por clima
- factor_ruta = 1.0 + ajustes por tipo de vía
- factor_mantenimiento = 1.0 + ajustes por rotación/align/balance

2. Genera lecturas cada 30 segundos
3. Simula rutas reales por carreteras NHS
4. Inyecta escenarios automáticamente:
   - Min 0-10: Operación normal
   - Min 10-15: Desgaste gradual en FL-1 (baja presión 1 psi/h)
   - Min 15-16: Pinchazo súbito en RR-2 (caída 30 psi en 1 min)
   - Min 16-20: Sobrecalentamiento en RL-3 (temp sube a 180°F)
   - Min 20-25: Frenazos bruscos (acelerómetro -2G)
   - Min 25-30: Sensores offline (TL-2 sin comunicación)
   - Min 30-35: Violación HOS (conductor lleva 12h manejando)
   - Min 35-40: Overweight (peso excesivo en eje drive)
   - Repetir ciclo

**Reglas de generación realista:**
- Presión base: 100 psi ± 5 psi (varía por posición)
- Temperatura base: 110°F + (speed_mph * 0.2) + random(-5, 5)
- Vibración: 0.2G + (speed_mph * 0.005) + random(-0.1, 0.1)
- Correlación entre llantas del mismo eje
- Efecto de velocidad y carga

**Modos:**
- `python simulador_iot.py --modo demo` (10 tractomulas, ciclo 40 min)
- `python simulador_iot.py --modo stress --tractomulas 1000`
- `python simulador_iot.py --escenario pinchazo --tractomula TRK-001`

**Logs con colores:**
