# Documentacion de PIN Platform

La carpeta `docs/` se organiza en dos bloques para que la navegacion sea clara en GitHub y durante la entrega:

## 01_entrega

Documentos principales usados para instalacion, presentacion y soporte del proyecto:

- `MEMORIA_FINAL_PRESENTACION.md`
- `MANUAL_INSTALACION_LOCAL.md`
- `INFORME_BASE_DATOS.md`
- `FUENTES_DATASETS.md`
- `ANEXO_E_TABLA_DATASETS.md`
- `METODOLOGIA.md`

## 02_fases

Trazabilidad historica del desarrollo, organizada por etapas del proyecto:

- `FASE_1_Estructura_Base.md`
- `FASE_2_App_Core.md`
- `FASE_3_App_Devices.md`
- `FASE_4_App_Tires_y_Mapa.md`
- `FASE_5_App_HOS.md`
- `FASE_6_App_Weight_Monitoring.md`
- `FASE_7_App_Fleet.md`
- `FASE_8_Apps_Criticas.md`

- `FASE_10_Simulador_IoT.md`
- `FASE_11_Dashboards_Web.md`
- `FASE_12_Data_Ingestion_ETL.md`
- `FASE_13_API_REST_Documentacion.md`
- `FASE_14_Documentacion_Tecnica.md`
- `FASE_15_Comandos_y_Scripts_Finales.md`

## Criterio de orden

- `01_entrega` contiene lo que un lector necesita primero para entender, instalar y presentar el proyecto.
- `02_fases` conserva la historia tecnica y academica de implementacion.
